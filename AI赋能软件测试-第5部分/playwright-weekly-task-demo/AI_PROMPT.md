# 给 AI 的课堂演示提示词

你是一名资深测试开发工程师。请使用 TypeScript 和 `@playwright/test`，为下面的业务场景生成一份可运行的 UI 自动化测试脚本。

## 测试环境

- 系统地址：`https://autotest.wiac.xyz/`
- 登录邮箱：通过环境变量 `AUTOTEST_EMAIL` 读取
- 登录密码：通过环境变量 `AUTOTEST_PASSWORD` 读取

## 测试目标

登录系统，在“任务管理”页面新建一个定时任务，并设置为每周一触发。若页面要求填写执行时间，默认设置为 `09:00`。

## 操作步骤

1. 打开系统登录页。
2. 输入邮箱和密码并登录。
3. 验证登录成功，不允许只使用固定等待时间。
4. 进入“任务管理”页面。
5. 点击“新建任务”或语义相同的按钮。
6. 输入唯一任务名称：`Playwright课堂演示-每周一任务-${Date.now()}`。
7. 将触发方式设置为“定时触发”。
8. 将执行周期设置为“每周”。
9. 选择“周一”。
10. 若存在执行时间字段，填写 `09:00`。
11. 保存任务。
12. 验证页面出现创建成功提示，并验证任务列表中出现该任务名称。
13. 保存成功后的页面截图。

## 编码要求

1. 使用 Playwright Test，不使用 Selenium。
2. 优先使用 `getByRole`、`getByLabel`、`getByPlaceholder`，不要优先使用长 XPath、动态 class 或 `nth()`。
3. 不得使用 `waitForTimeout()` 作为主要等待方式。
4. 账号密码必须通过 `.env` 读取，不得直接写死在测试代码中。
5. 使用 `test.step()` 划分登录、导航、填写表单、保存和断言步骤。
6. 配置失败截图、Trace 和录像。
7. 对自定义下拉框兼容 Ant Design 或 Element Plus 的常见 DOM 结构。
8. 输出完整项目文件：`package.json`、`playwright.config.ts`、`.env.example`、测试脚本和运行说明。
9. 对无法从需求确定的必填字段，用清晰的 `TODO` 标记，不得虚构业务值。
10. 生成代码后，再给出一份人工审查清单，重点检查业务路径、定位器、断言、测试数据和可重复执行性。
