# 分类检查清单

## 停线规则

当发生任何意外情况时：

```
1. 停止添加功能或进行更改
2. 保留证据（错误输出、日志、复现步骤）
3. 使用分类检查清单诊断
4. 修复根本原因
5. 防止再次发生
6. 仅在验证通过后恢复
```

**不要跳过失败的测试或中断的构建去处理下一个功能。** 错误会累积。步骤 3 中未修复的 bug 会使步骤 4-10 错误。

## 分类检查清单

按顺序处理这些步骤。不要跳过步骤。

### 步骤 1：复现

使失败可靠地发生。如果无法复现，就无法有信心地修复。

```
你能复现失败吗？
├── 是 → 继续步骤 2
└── 否
    ├── 收集更多上下文（日志、环境详细信息）
    ├── 尝试在最小环境中复现
    └── 如果确实无法复现，记录条件并监控
```

**当 bug 无法复现时：**

```
无法按需复现：
├── 依赖时序？
│   ├── 在可疑区域添加时间戳到日志
│   ├── 尝试使用人工延迟（setTimeout、sleep）扩大竞争窗口
│   └── 在负载或并发下运行以增加碰撞概率
├── 依赖环境？
│   ├── 比较 Node/浏览器版本、操作系统、环境变量
│   ├── 检查数据差异（空数据库 vs 已填充数据库）
│   └── 尝试在环境干净的 CI 中复现
├── 依赖状态？
│   ├── 检查测试或请求间泄漏的状态
│   ├── 查找全局变量、单例或共享缓存
│   └── 在隔离 vs 其他操作后运行失败场景
└── 真正随机？
    ├── 在可疑位置添加防御性日志
    ├── 为特定错误签名设置警报
    └── 记录观察到的条件并在再次发生时重新检查
```

对于测试失败：
```bash
# 运行特定失败测试
npm test -- --grep "测试名称"

# 运行详细输出
npm test -- --verbose

# 隔离运行（排除测试污染）
npm test -- --testPathPattern="特定文件" --runInBand
```

### 步骤 2：定位

缩小失败发生的位置：

```
哪个层失败？
├── UI/前端     → 检查控制台、DOM、网络标签页
├── API/后端    → 检查服务器日志、请求/响应
├── 数据库      → 检查查询、模式、数据完整性
├── 构建工具    → 检查配置、依赖、环境
├── 外部服务    → 检查连接性、API 变更、速率限制
└── 测试本身    → 检查测试是否正确（假阴性）
```

**对回归 bug 使用二分法：**
```bash
# 查找哪个提交引入了 bug
git bisect start
git bisect bad                    # 当前提交损坏
git bisect good <已知良好提交>     # 此提交工作
# Git 将检出中间提交；在每个上运行测试
git bisect run npm test -- --grep "失败测试"
```

### 步骤 3：缩减

创建最小失败案例：
- 移除不相关的代码/配置，直到只剩下 bug
- 将输入简化为触发失败的最小示例
- 将测试缩减为复现问题的最低限度

最小复现使根本原因显而易见，并防止修复症状而非原因。

### 步骤 4：修复根本原因

修复根本问题，而非症状：

```
症状："用户列表显示重复条目"

症状修复（错误）：
  → 在 UI 组件中去重：[...new Set(users)]

根本原因修复（正确）：
  → API 端点有一个产生重复的 JOIN
  → 修复查询，添加 DISTINCT，或修复数据模型
```

问："为什么会发生这种情况？"直到找到实际原因，而不仅仅是它表现的地方。

### 步骤 5：防止再次发生

编写捕获此特定失败的测试：

```typescript
// bug：包含特殊字符的任务标题破坏了搜索
it('查找标题中包含特殊字符的任务', async () => {
  await createTask({ title: 'Fix "quotes" & <brackets>' });
  const results = await searchTasks('quotes');
  expect(results).toHaveLength(1);
  expect(results[0].title).toBe('Fix "quotes" & <brackets>');
});
```

此测试将防止相同的 bug 再次发生。它应该在没有修复时失败，在有修复时通过。

### 步骤 6：端到端验证

修复后，验证完整场景：

```bash
# 运行特定测试
npm test -- --grep "特定测试"

# 运行完整测试套件（检查回归）
npm test

# 构建项目（检查类型/编译错误）
npm run build

# 手动抽查（如适用）
npm run dev  # 在浏览器中验证
```
