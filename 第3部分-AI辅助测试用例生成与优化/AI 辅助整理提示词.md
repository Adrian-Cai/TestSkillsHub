## 用例裁剪与合并清单

### AI 辅助整理提示词

```text
请帮我整理以下测试用例。

请识别：
1. 重复用例
2. 相似用例
3. 粒度过细的用例
4. 粒度过粗的用例
5. 不符合业务规则的用例
6. 执行价值较低的用例

请输出：
- 建议删除的用例及原因
- 建议合并的用例及合并方式
- 建议拆分的用例及拆分方式
- 建议保留的高价值用例

注意：
- 只给出整理建议，不要直接删除
- 对涉及业务判断的内容标记“需人工确认”

测试用例如下：
【粘贴测试用例】
```

### 裁剪与合并判断表

| 判断项 | 处理建议 |
|---|---|
| 用例验证目标完全相同 | 合并 |
| 只有测试数据不同，但验证逻辑相同 | 合并为参数化用例 |
| 业务场景不成立 | 删除 |
| 与当前版本无关 | 移至后续版本或回归池 |
| 只验证低价值展示内容 | 降低优先级或合并 |
| 一个用例覆盖多个独立风险 | 拆分 |
| 步骤过长且失败定位困难 | 拆分 |
| 前置条件复杂但风险低 | 降低优先级 |
| 涉及核心链路或资损风险 | 保留 |
| 涉及历史高发缺陷 | 保留并提高优先级 |
