# 产物文件输出规范

## 默认输出位置

每次执行必须生成 Markdown 文件：

```text
outputs/testcases/<slug>.md
```

`<slug>` 使用功能名称、需求名称或用户提供的主题，转成简短英文或拼音标识。

## 对话回复要求

最终回复只做摘要，不要把完整产物重复粘贴到对话中。

回复必须包含：

- 产物文件路径。
- 已覆盖的主要内容。
- 如果存在待确认问题，提示用户查看对应章节。

## 文件内容要求

产物必须包含：

- 需求与范围摘要。
- 业务规则拆解。
- 测试设计方法覆盖。
- 测试用例初稿。
- 遗漏检查与补充场景。
- 裁剪与合并建议。
- 最终测试用例集。
- 优先级说明。
- 待确认问题。

## 不允许

- 不允许只输出零散测试点。
- 不允许只输出 AI 直接生成的用例表。
- 不允许省略人工审查、遗漏检查、裁剪合并和优先级排序。
- 不允许把不确定业务规则写成确定结论。
