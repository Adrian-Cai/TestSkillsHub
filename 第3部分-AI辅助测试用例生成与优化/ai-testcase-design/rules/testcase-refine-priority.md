# 用例裁剪、合并与优先级规则

## 裁剪与合并规则

| 判断项 | 处理建议 |
| --- | --- |
| 验证目标完全相同 | 合并 |
| 只有测试数据不同，验证逻辑相同 | 合并为参数化用例 |
| 业务场景不成立 | 删除 |
| 与当前版本无关 | 移至待确认或后续版本 |
| 只验证低价值展示内容 | 降低优先级或合并 |
| 一个用例覆盖多个独立风险 | 拆分 |
| 步骤过长且失败定位困难 | 拆分 |
| 前置条件复杂但风险低 | 降低优先级 |
| 涉及核心链路或资损风险 | 保留 |
| 涉及历史高发缺陷 | 保留并提高优先级 |

## 优先级判断

| 优先级 | 判断标准 |
| --- | --- |
| P0 | 核心链路、上线阻断、资损、权限越界、数据严重错误 |
| P1 | 重要分支、常见异常、影响主要用户体验 |
| P2 | 低频场景、展示类问题、影响范围较小 |

## 输出格式

```markdown
## 裁剪与合并建议
| 原用例 | 处理方式 | 原因 | 合并后用例 |
| --- | --- | --- | --- |

## 最终测试用例集
| 用例编号 | 用例标题 | 覆盖目标 | 前置条件 | 步骤 | 预期结果 | 优先级 |
| --- | --- | --- | --- | --- | --- | --- |
```

## 责任边界

- AI 可以建议删除、合并、排序。
- 测试人员必须确认最终测试范围。
- 高风险用例的优先级理由必须写清楚。
