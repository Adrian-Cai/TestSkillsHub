## 测试设计方法 生成用例提示词模板

### 通用模板

```text
请基于以下已经确认的业务规则，使用测试设计方法生成测试用例初稿。

请分别按照以下方法生成：
1. 场景法：覆盖主流程、分支流程、异常流程
2. 等价类：划分有效等价类和无效等价类
3. 边界值：识别金额、时间、次数、长度、数量等边界
4. 判定表：整理多条件组合下的系统行为
5. 状态迁移法：识别状态、事件、动作、合法流转和非法流转

每条用例请包含：
- 用例编号
- 用例标题
- 测试设计方法
- 前置条件
- 测试数据
- 操作步骤
- 预期结果
- 初始优先级建议

要求：
- 不要生成与业务规则无关的用例
- 不要重复生成相同验证目标的用例
- 对不确定规则标记“需确认”
- 优先覆盖核心流程、高风险场景、异常场景和边界场景

业务规则如下：
【粘贴业务规则拆解清单】
```

### 判定表专项模板

```text
请基于以下业务规则，为该功能设计判定表。

请输出：
1. 条件项
2. 动作项
3. 条件组合
4. 每种组合下的系统行为
5. 根据判定表提炼出的测试用例

要求：
- 合并不可能出现的条件组合
- 标记需要业务确认的组合
- 优先覆盖容易造成错误结果或资损的组合

业务规则如下：
【粘贴业务规则】
```

### 状态迁移专项模板

```text
请基于以下业务规则，分析该功能的状态迁移。

请输出：
1. 状态列表
2. 触发事件
3. 合法状态迁移
4. 非法状态迁移
5. 每个状态迁移对应的测试用例

要求：
- 重点关注状态变更后的数据一致性
- 重点关注重复操作、过期操作、取消操作、回退操作
- 对需求没有说明的状态规则标记“需确认”

业务规则如下：
【粘贴业务规则】
```
