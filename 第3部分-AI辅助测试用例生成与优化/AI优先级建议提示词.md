##  用例优先级判断清单

### AI 优先级建议提示词

```text
请基于以下测试用例，给出 P0 / P1 / P2 优先级建议。

判断依据：
1. 是否覆盖核心业务链路
2. 是否涉及资损、金额、订单、账户、权益
3. 是否涉及权限、越权、数据泄露
4. 是否为用户高频操作
5. 是否影响范围大
6. 是否属于历史缺陷高发区域
7. 是否为上线阻断风险
8. 是否执行成本高但风险较低

请输出：
- 用例编号
- 建议优先级
- 优先级理由
- 是否建议纳入冒烟测试
- 是否需要人工确认

测试用例如下：
【粘贴测试用例】
```

### 优先级判断标准

| 优先级 | 判断标准 | 示例 |
|---|---|---|
| P0 | 核心链路、上线阻断、资损、权限越界、数据严重错误 | 优惠券可重复使用、过期券可使用、金额计算错误 |
| P1 | 重要分支、常见异常、影响主要用户体验 | 未达到门槛不可用、商品不适用不可用、订单取消后券状态正确 |
| P2 | 低频场景、展示类问题、影响范围较小 | 优惠券列表排序、提示文案、非核心筛选条件 |

### 人工确认问题

| 问题 | 确认对象 | 结论 |
|---|---|---|
| 当前版本是否覆盖订单取消退券规则 | 产品 / 业务 |  |
| 是否存在不同用户角色或会员等级 | 产品 / 业务 |  |
| 是否需要覆盖并发领取或并发使用 | 开发 / 架构 |  |
| 优惠券金额计算是否涉及多优惠叠加 | 产品 / 开发 |  |
