# 功能测试用例生成模板

## 使用场景

当已经完成需求摘要、业务规则清单和测试点矩阵后，使用本模板让 AI 生成结构化功能测试用例初稿。

## 输入材料

- 需求摘要
- 业务规则清单
- 测试点矩阵
- 需求疑问与风险清单
- 项目已有用例格式，如有

## Prompt 模板

```text
你是一名资深功能测试工程师。

请基于以下材料，生成结构化功能测试用例初稿。

【需求摘要】
<粘贴需求摘要>

【业务规则清单】
<粘贴业务规则清单>

【测试点矩阵】
<粘贴测试点矩阵>

【需求疑问与风险清单】
<粘贴风险清单，如没有则填写“暂无”>

请按以下字段输出测试用例：
| 用例编号 | 模块 | 测试场景 | 前置条件 | 测试数据 | 操作步骤 | 预期结果 | 优先级 | 关联规则/风险 |

要求：
1. 用例必须来源于测试点矩阵，不要凭空新增与需求无关的用例。
2. 每条用例要有清晰前置条件、测试数据、操作步骤和预期结果。
3. 正常流、异常流、边界值、高风险场景必须覆盖。
4. 对需求未明确的内容，不要写成确定预期结果，请标记为“待确认”。
5. 优先级使用 P0 / P1 / P2 / P3。
6. 输出后再补充一段“人工审查建议”，指出哪些用例需要测试人员重点确认。
```

## 用例字段说明

| 字段 | 填写说明 |
| --- | --- |
| 用例编号 | 建议格式：模块缩写-序号，如 COUPON-001 |
| 模块 | 功能模块或子模块 |
| 测试场景 | 一句话描述测试目标 |
| 前置条件 | 账号、数据、配置、状态等前提 |
| 测试数据 | 金额、时间、优惠券状态、商品范围等 |
| 操作步骤 | 可执行、可复现、按顺序描述 |
| 预期结果 | 可观察、可验证，不写模糊结果 |
| 优先级 | P0 最高，P3 最低 |
| 关联规则/风险 | 对应业务规则或风险点 |

## 人工审查清单

- 用例是否都来自已确认的规则和测试点
- 前置条件是否真实可构造
- 测试数据是否覆盖边界值
- 操作步骤是否可执行
- 预期结果是否可观察
- 高风险场景是否标记为 P0 / P1
- 待确认需求是否没有被写成确定结论

## 最终产出

- 功能测试用例初稿
- 待确认用例清单
- 高风险用例清单
- 可补充到测试计划中的优先级建议
