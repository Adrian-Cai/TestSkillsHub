## 2. AI需求摘要检查

| 检查项 | 人工审查重点 |
| --- | --- |
| 业务目标是否准确 | 是否偏离真实业务目的 |
| 用户角色是否完整 | 是否遗漏后台、运营、管理员等角色 |
| 功能范围是否清楚 | 是否把未提及内容误认为本次范围 |
| 主流程是否完整 | 是否覆盖从入口到结果的完整链路 |
| 前置条件是否明确 | 是否需要登录、权限、配置、状态 |
| 待确认内容是否标出 | 不确定信息不能被 AI 当成事实 |

## 3. AI规则提取检查

| 规则类型 | 检查问题 |
| --- | --- |
| 业务规则 | 触发条件、成功条件、失败条件是否明确 |
| 字段规则 | 必填、格式、长度、默认值、枚举值是否明确 |
| 权限规则 | 不同角色、不同状态下是否允许操作 |
| 状态规则 | 状态流转是否完整，是否有终态和回退规则 |
| 时间规则 | 有效期、超时、开始时间、结束时间是否明确 |
| 金额 / 数量规则 | 门槛、上限、下限、次数、精度是否明确 |
| 异常处理 | 错误提示、失败回滚、重试逻辑是否明确 |

## 5. 人工审查结论

| 审查结论 | 说明 |
| --- | --- |
| 可进入用例设计 | 需求规则清晰，测试点已基本完整 |
| 需补充需求说明 | 存在规则缺口或歧义，需产品确认 |
| 需技术确认 | 涉及状态、接口、数据一致性或异常回滚 |
| 需风险升级 | 存在高业务影响或高概率失败场景 |
