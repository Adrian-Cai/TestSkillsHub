# 本章可交付物清单

## 适用章节

AI 辅助需求评审与测试点设计

## 课堂演示产物

| 交付物 | 用途 | 使用阶段 |
| --- | --- | --- |
| 需求摘要示例 | 将长需求整理为测试人员可理解的业务摘要 | 需求理解 |
| 业务规则清单示例 | 将散落描述拆成可审查的规则 | 需求评审 |
| 优惠券功能测试点矩阵示例 | 展示从规则到测试点的拆解方式 | 测试点设计 |
| 需求疑问与风险清单示例 | 暴露需求歧义、缺口和高风险场景 | 评审会议 |
| 风险优先级排序示例 | 帮助确定测试投入顺序 | 测试计划 |

## 课后复用材料

| 交付物 | 复用方式 |
| --- | --- |
| 《AI辅助需求评审清单》 | 每次需求评审前后检查需求是否可测、规则是否完整 |
| 《测试点拆解 Prompt 模板》 | 将需求摘要和规则清单交给 AI，生成测试点矩阵初稿 |
| 《功能测试用例生成模板》 | 基于测试点矩阵生成结构化功能测试用例 |
| 《优惠券功能测试点矩阵示例》 | 作为课堂案例和团队模板参考 |
| 《需求疑问与风险清单示例》 | 作为评审会议前的问题整理样例 |

## 使用提醒

- AI 输出是初稿，不是最终测试方案。
- 测试人员负责确认业务规则、风险优先级和最终质量。
- 所有 AI 生成内容都要经过人工审查后再进入测试设计或评审材料。
