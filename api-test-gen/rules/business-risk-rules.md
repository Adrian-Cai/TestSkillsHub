# 业务风险识别规则

## 状态流转类接口
适用场景：
- 订单状态
- 活动状态
- 审核状态
- 任务状态

必须补充测试：
- 初始状态是否合法
- 状态是否允许跳转
- 重复操作是否幂等
- 非法状态是否拒绝
- 状态变更后数据库是否一致
- 前端展示状态是否同步
- 是否产生异步消息
- 是否影响下游链路

## 资金 / 积分 / 奖励类接口
必须补充测试：
- 金额 / 积分是否精度正确
- 是否允许重复发放
- 是否存在并发重复领取
- 是否有流水记录
- 是否可追溯
- 回滚 / 补偿是否正确
- 新老系统双写是否一致

## 灰度 / 配置类接口
必须补充测试：
- 开关开启
- 开关关闭
- 灰度用户
- 非灰度用户
- 配置变更后老数据兼容
- 降级逻辑
- 日志是否可观测
