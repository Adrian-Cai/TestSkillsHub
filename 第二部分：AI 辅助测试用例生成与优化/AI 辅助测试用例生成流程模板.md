## AI 辅助测试用例生成流程模板

### 流程总览

| 步骤 | 输入 | AI 处理 | 人工审查 | 最终产出 |
|---|---|---|---|---|
| 1. 需求输入 | 需求说明、原型、用户故事、接口说明、业务规则 | 读取并总结需求信息 | 判断需求范围是否完整 | 需求输入材料包 |
| 2. 业务规则拆解 | 原始需求材料 | 提取角色、对象、规则、限制、状态、异常、疑问点 | 检查 AI 是否误解业务，补充遗漏规则 | 业务规则拆解清单 |
| 3. 方法化生成用例 | 已确认的业务规则 | 按场景法、等价类、边界值、判定表、状态迁移法生成用例 | 判断场景是否成立，删除不符合业务实际的内容 | 测试用例初稿 |
| 4. 反向检查遗漏 | 原始需求 + 用例初稿 | 从异常、边界、权限、状态、并发、数据一致性等角度检查 | 判断遗漏是否真实、是否值得覆盖 | 遗漏场景补充清单 |
| 5. 裁剪与合并 | 用例初稿 + 补充场景 | 标识重复、相似、粒度异常的用例 | 删除、合并、调整粒度 | 可执行测试用例集 |
| 6. 优先级排序 | 可执行测试用例集、业务风险、历史缺陷 | 给出 P0/P1/P2 建议 | 结合版本目标、资源和风险最终确认 | 带优先级的最终用例集 |

### 推荐用例字段

| 字段 | 说明 |
|---|---|
| 用例编号 | 便于追踪和评审 |
| 用例标题 | 用一句话表达测试目标 |
| 测试设计方法 | 场景法 / 等价类 / 边界值 / 判定表 / 状态迁移法 |
| 前置条件 | 执行前必须满足的条件 |
| 测试数据 | 关键输入数据 |
| 操作步骤 | 控制在可执行粒度 |
| 预期结果 | 可验证、可判断 |
| 优先级 | P0 / P1 / P2 |
| 备注 | 记录风险、依赖或待确认点 |
