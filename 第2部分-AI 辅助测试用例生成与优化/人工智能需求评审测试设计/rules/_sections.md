# 规则导航

执行 `ai-requirement-review-test-design` 时，先阅读本文件，再按任务需要读取其他规则文件。

## 必读
- `artifact-file-output.md`：产物路径、命名和最终回复要求。
- `requirement-review-workflow.md`：需求输入到测试产出的闭环流程。

## 按需读取
- `test-point-dimensions.md`：拆解测试点时读取。
- `risk-priority-rules.md`：需要排序风险或标注优先级时读取。
- `prompt-template-rules.md`：需要输出 Prompt 模板或课程交付物时读取。

## 输出优先级
默认优先输出：
1. 需求摘要
2. 业务规则清单
3. 测试点矩阵
4. 需求疑问与风险清单
5. AI辅助需求评审清单
6. Prompt 模板
