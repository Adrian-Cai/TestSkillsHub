# Prompt 模板规则

输出 Prompt 模板时，模板必须服务测试工作流，而不是泛泛提问。

## 测试点拆解 Prompt 必须包含
- 输入：需求文档或功能说明
- AI任务：摘要、规则、正常流、异常流、边界值、隐含需求、测试点矩阵
- 约束：不确定内容进入待确认项
- 输出格式：表格字段明确
- 人工审查：要求标注需要测试人员确认的地方

## 功能测试用例生成 Prompt 必须包含
- 输入：测试点矩阵
- AI任务：生成用例标题、前置条件、步骤、数据、预期结果
- 约束：规则不明时不要编造预期
- 输出格式：可直接进入用例管理工具

## 禁止写法
- “帮我生成完整测试用例”这类过宽 Prompt。
- 只要求数量，不要求规则依据。
- 让 AI 直接替代人工确认。
