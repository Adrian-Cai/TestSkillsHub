# 需求评审闭环

本技能必须体现闭环：需求输入 -> AI处理 -> 人工审查 -> 测试产出。

## 输入
识别需求来源、业务目标、功能范围、用户角色、前置条件、主流程和依赖系统。

## AI处理
让 AI 做初稿和结构化整理：
- 生成需求摘要
- 提取规则清单
- 扩展正常流、异常流、边界值
- 识别隐含需求和歧义
- 生成测试点矩阵

## 人工审查
明确提醒测试人员审查：
- AI 是否误解业务语义
- 规则是否有依据
- 边界口径是否需要产品确认
- 风险排序是否符合业务实际
- 是否存在资金、权限、数据一致性风险

## 测试产出
最终产出必须能被测试团队继续使用：
- 评审清单可用于需求评审会议
- 测试点矩阵可继续生成测试用例
- 风险清单可用于评估测试优先级
- Prompt 模板可复用到同类需求
