## 用例遗漏检查清单

### AI 反向检查提示词

```text
请作为资深测试负责人，检查以下测试用例是否存在遗漏。

请结合原始需求，从以下维度检查：
1. 主流程是否完整
2. 分支流程是否完整
3. 异常输入是否覆盖
4. 边界值是否覆盖
5. 权限和角色差异是否覆盖
6. 状态流转是否覆盖
7. 重复操作是否覆盖
8. 并发或重复提交是否覆盖
9. 数据为空、重复、失效、过期是否覆盖
10. 数据一致性是否覆盖
11. 取消、回退、撤销、恢复等逆向流程是否覆盖
12. 是否存在重复用例或粒度不一致问题

请输出：
- 遗漏点
- 建议补充的测试场景
- 建议合并或删除的用例
- 需要人工确认的问题

原始需求：
【粘贴需求】

现有测试用例：
【粘贴用例初稿】
```

### 人工审查清单

| 检查维度 | 人工审查问题 | 是否通过 |
|---|---|---|
| 主流程 | 是否覆盖用户最常用、最核心的业务路径 | 是 / 否 |
| 分支流程 | 是否覆盖不同业务条件下的分支行为 | 是 / 否 |
| 异常输入 | 是否覆盖空值、非法值、格式错误、超范围输入 | 是 / 否 |
| 边界值 | 是否覆盖等于边界、刚小于边界、刚大于边界 | 是 / 否 |
| 权限角色 | 是否覆盖不同角色、无权限、越权操作 | 是 / 否 |
| 状态流转 | 是否覆盖合法状态变化和非法状态变化 | 是 / 否 |
| 重复操作 | 是否覆盖重复提交、重复领取、重复支付、重复使用 | 是 / 否 |
| 并发场景 | 是否覆盖并发点击、并发使用、库存或额度竞争 | 是 / 否 |
| 数据一致性 | 前端、后端、订单、账户、日志状态是否一致 | 是 / 否 |
| 逆向流程 | 是否覆盖取消、回退、撤销、恢复 | 是 / 否 |
| 用例质量 | 是否存在重复、过细、过粗、不可执行用例 | 是 / 否 |
