# 参考答案与讲评要点

本文件不是标准答案，用于教师备课和演示兜底。

## 一、需求摘要参考

- 业务目标：通过新人优惠券提升新用户首次下单转化率。
- 用户角色：已完成手机号注册、符合“新用户”条件的商城用户。
- 功能范围：优惠券领取、展示、选择、抵扣、核销及异常状态处理。
- 主流程：注册新用户领取优惠券 → 选择符合条件的商品 → 提交满门槛订单 → 选择优惠券 → 抵扣 10 元 → 支付成功后完成核销。

## 二、明确规则参考

| 编号 | 规则 | 明确程度 |
|---|---|---|
| R01 | 完成手机号注册的新用户可领取 | 明确，但“新用户”口径待确认 |
| R02 | 每个用户只能领取一次 | 明确 |
| R03 | 面额为 10 元 | 明确 |
| R04 | 领取后 7 天内有效 | 时长明确，时间计算口径待确认 |
| R05 | 订单金额满 100 元可使用 | 门槛明确，金额计算口径待确认 |
| R06 | 每笔订单只能使用一张 | 明确 |
| R07 | 仅适用于普通商品，不适用于秒杀商品 | 明确，混合订单口径待确认 |
| R08 | 不能与其他优惠券同时使用 | 明确 |
| R09 | 已过期、已使用的优惠券不能再次使用 | 明确 |
| R10 | 支付失败时应正确处理优惠券状态 | 处理要求不明确 |

## 三、优先进入评审的需求疑问

| 问题 | 影响 |
|---|---|
| 新用户如何定义？注销重注册、换手机号、已有订单用户如何处理？ | 领取资格和防刷规则 |
| 满 100 元按原价、实付价还是适用商品金额计算？ | 优惠资格与金额计算 |
| 领取后 7 天的起止时间及边界如何计算？ | 到期判断和用户投诉 |
| 普通商品与秒杀商品混合下单时如何计算门槛？ | 组合订单优惠资格 |
| 支付失败、支付超时后何时恢复优惠券？ | 券状态一致性和用户再次下单 |
| 订单取消或退款后优惠券是否退回？ | 售后流程和资金营销成本 |
| 后台可以配置哪些规则？配置变更何时生效？ | 前后台一致性及存量券处理 |

## 四、高价值测试点参考

| 关联规则 | 测试点 | 类型 | 优先级 |
|---|---|---|---|
| R05 | 订单金额分别为 99.99、100、100.01 时的可用性 | 边界值 | 高 |
| R04 | 有效期结束前、结束时刻、结束后的可用状态 | 时间边界 | 高 |
| R02 | 同一用户重复领取 | 异常/幂等 | 高 |
| R07 | 仅普通商品、仅秒杀商品、混合商品订单 | 组合场景 | 高 |
| R08 | 同一订单尝试选择两张优惠券 | 规则校验 | 中 |
| R09 | 已核销优惠券再次使用 | 状态校验 | 高 |
| R10 | 支付失败后优惠券状态恢复 | 状态变化 | 高 |
| R10 | 支付超时、回调延迟时优惠券状态 | 异常/一致性 | 高 |
| R05/R10 | 重复提交订单时不得重复占用或核销 | 并发/幂等 | 高 |
| R03 | 抵扣后订单金额、优惠明细和支付金额一致 | 数据一致性 | 高 |

## 五、AI 容易出现的错误

- 默认优惠券可与积分、满减或会员折扣叠加。
- 默认取消订单或退款后一定退券。
- 默认优惠券从领取时刻开始精确计算 7×24 小时。
- 默认“订单金额”就是最终实付金额。
- 自动补充需求中没有出现的渠道、店铺、地区或会员等级限制。
- 给出确定预期，但没有暴露规则本身尚未明确。

## 六、讲评结论

判断 AI 输出是否可用，不看条数，主要看四点：

1. 是否忠于需求原文；
2. 是否把未知内容标为待确认；
3. 测试点是否能追溯到规则；
4. 优先级是否结合真实业务背景重新判断。
