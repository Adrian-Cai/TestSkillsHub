# Phase 2 产物示例：配图 Prompt

> 对应 `plan/ai-testing-engineer-survival-plan.md` 的 6 个核心章节。本文档展示"达标"的封面 + 6 张正文配图 Prompt（中英双语）。
>
> 严格遵守 `visuals-body-prompt.md` 的 design token：主色 `#0f2a5f`、强调色 `#14b8a6`、白底、4:3 比例、信息图 / 架构图 / 流程图 / 白板草图风格。

---

## 【微信公众号封面图 Prompt】

**比例**：2.35:1
**章节名**：AI 时代测试工程师的价值重估
**风格**：AI 测试技术风、专业、克制、简洁、科技感
**颜色**：深蓝 + 青绿色为主
**画面气质**：干净、高级、理性，不营销、不夸张
**构图**：适合公众号封面，留有标题空间

**中文**：
公众号封面图，2.35:1 比例，白底主调，中心区域是简洁的"价值曲线"图示：一条由低到高再下行的折线，标注"测试能力"和"AI 时代"。主色深蓝 `#0f2a5f` + 强调色青绿 `#14b8a6`。曲线下方有一只手在某个转折点做标记（手部线稿风格），象征"重新定价"。留出上方 1/3 区域作为标题文字位。整体保持技术感、克制、干净，无任何人物面部、无任何营销元素。

**英文**：
WeChat official-account cover image, 2.35:1 aspect ratio, white background. Center: a clean "value curve" infographic — a single line going up then plateauing, labeled "Testing Capability" and "AI Era". Color palette: deep blue `#0f2a5f` and teal `#14b8a6`. A schematic hand (line art) marks a pivot point on the curve, symbolizing re-pricing. Upper third reserved for headline text. Tech, restrained, clean. No faces, no marketing elements.

## 【正文配图 Prompt 1】

**章节名**：我在团队里看到的两个测试人
**信息类型**：场景对比

**中文**：
4:3 比例信息图，白底。画面分成左右两栏：左侧标签"测试人 A"，画一个简洁线稿人物站在一组重复性文档堆旁（图标：清单 / 模板），标注"重复性用例编写"；右侧标签"测试人 B"，画同样线稿人物站在一张白板前（图标：白板 + 风险词条），标注"风险建模 / 业务对话"。主色 `#0f2a5f`，强调色 `#14b8a6` 用于连线与关键词。整体是测试工程师在技术分享时画的精致白板图，轻微手绘感。

**英文**：
4:3 infographic, white background. Two-column comparison: left column "Tester A" with a simple line-art figure beside a stack of repetitive documents (icons: checklists, templates), labeled "Repetitive Test Case Writing". Right column "Tester B" with a similar figure standing in front of a whiteboard (icons: whiteboard + risk keywords), labeled "Risk Modeling / Business Conversations". Color: deep blue `#0f2a5f` and teal `#14b8a6` for connectors and keywords. Whiteboard-style sketch, slightly hand-drawn but professional.

## 【正文配图 Prompt 2】

**章节名**：AI 真正改变的是什么：不是"做什么"
**信息类型**：概念区分

**中文**：
4:3 比例架构图，白底。中心放两个并列的圆形节点：左侧"做什么"（包含一组重复的齿轮图标），右侧"值多少钱"（包含一个天平 + 货币符号）。用强调色 `#14b8a6` 画一条贯穿两者的箭头，标注"AI 改变的是这条箭头"。上方有一个浅色 banner："AI 改变的不是动作，而是价值"。线稿风格，干净技术感。

**英文**：
4:3 architecture diagram, white background. Two parallel circular nodes: left "What you do" (with repetitive gear icons), right "What it's worth" (with a balance scale + currency symbol). A teal `#14b8a6` arrow runs through both, labeled "What AI changes". A subtle banner above: "AI changes the value, not the action". Line-art, clean, tech feel.

## 【正文配图 Prompt 3】

**章节名**：测试能力价值曲线：哪些在涨、哪些在跌
**信息类型**：趋势曲线

**中文**：
4:3 比例图表，白底。画一个价值曲线图：横轴是"AI 渗透阶段"（从 0% 到 100%），纵轴是"能力价值"。两条曲线：
- 上升曲线（强调色 `#14b8a6`）：标签"风险建模 / 业务理解 / 跨团队沟通"，起点低、终点高
- 下降曲线（深蓝 `#0f2a5f`）：标签"重复用例编写 / 固定脚本维护 / 文档化测试"，起点中、终点低
两条曲线在中间位置交叉。曲线下方有阴影区表示"价值迁移方向"。白板图风格，标注清晰。

**英文**：
4:3 chart, white background. Value curve graph: x-axis "AI Penetration Stage" (0% to 100%), y-axis "Capability Value". Two curves:
- Rising curve (teal `#14b8a6`): "Risk Modeling / Business Understanding / Cross-team Communication"
- Declining curve (deep blue `#0f2a5f`): "Repetitive Case Writing / Fixed Script Maintenance / Documented Testing"
Curves cross at midpoint. Shaded area below indicates value migration. Whiteboard-style, clear labels.

## 【正文配图 Prompt 4】

**章节名**：三个被 AI 重新定价的能力
**信息类型**：分类清单

**中文**：
4:3 比例卡片式信息图，白底。三个并排卡片（从左到右）：
- 卡片 1："业务语义理解"（图标：对话气泡 + 流程图）
- 卡片 2："风险建模"（图标：风险矩阵 + 决策树）
- 卡片 3："协作与判断"（图标：多人 + 决策路径）
每张卡片下方用一个小型上升箭头（强调色 `#14b8a6`）表示"价值上升"。卡片有 1px 边框 `#dbeafe`，背景白色。白板图风格。

**英文**:
4:3 card-style infographic, white background. Three side-by-side cards:
- Card 1: "Business Semantics" (icons: speech bubble + flowchart)
- Card 2: "Risk Modeling" (icons: risk matrix + decision tree)
- Card 3: "Collaboration & Judgment" (icons: multiple figures + decision path)
Each card has a small rising arrow (teal `#14b8a6`) below indicating "value up". Cards have 1px border `#dbeafe`, white fill. Whiteboard-style.

## 【正文配图 Prompt 5】

**章节名**：升值的部分长什么样：风险建模
**信息类型**：流程图

**中文**：
4:3 比例流程图，白底。从左到右的 4 步骤流程：
1. 需求变更输入（图标：文档 + 红色问号）
2. 风险识别（图标：放大镜 + 风险矩阵）
3. 影响范围分析（图标：依赖图）
4. 灰度策略（图标：流量分配 + 监控仪表盘）
步骤间用主色 `#0f2a5f` 箭头连接，步骤名称用加粗文字，下方附"实际产出：可执行测试策略"的小标签。强调色 `#14b8a6` 用于高亮当前步骤（在第 2 步）。白板图风格。

**英文**:
4:3 flowchart, white background. Four steps left to right:
1. Requirement Change Input (icons: document + red question mark)
2. Risk Identification (icons: magnifying glass + risk matrix)
3. Impact Range Analysis (icons: dependency graph)
4. Canary Strategy (icons: traffic distribution + monitoring dashboard)
Arrows in deep blue `#0f2a5f`, step names in bold. Teal `#14b8a6` highlights the current step (Step 2). Small label below: "Output: executable test strategy". Whiteboard-style.

## 【正文配图 Prompt 6】

**章节名**：怎么判断自己在那条曲线上
**信息类型**：决策路径

**中文**：
4:3 比例决策树，白底。一个 Y 形决策树：
- 起点："上周你做的最有价值的事是什么？"
- 左分支："写了 N 条自动化用例" → 终点"贬值区：价值持平或下降"（深蓝 `#0f2a5f`）
- 右分支："帮团队识别了一个线上风险" → 终点"升值区：价值持续上升"（强调色 `#14b8a6`）
每个终点下方有 1 行小字解释为什么对应那条曲线。白板图风格，留白充足。

**英文**:
4:3 decision tree, white background. Y-shaped decision tree:
- Root: "What was your most valuable work last week?"
- Left branch: "Wrote N automated test cases" → "Depreciation Zone: value flat or declining" (deep blue `#0f2a5f`)
- Right branch: "Helped the team identify a production risk" → "Appreciation Zone: value rising" (teal `#14b8a6`)
Each endpoint has a one-line explanation. Whiteboard-style, ample whitespace.
