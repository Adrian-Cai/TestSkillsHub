# 原始主题输入（Phase 0 示例）

> 这是用户在第一轮对话中提供的"本期输入"。信息故意只给"主题 + 一句话方向"，不提供栏目、历史文章库、字数要求、目标读者等附加信息，用以模拟真实工作场景中的"用户只给最少信息"情况。

## 用户原话

```
主题：AI 不会淘汰测试工程师

方向：想写一篇关于 AI 时代测试工程师职业发展的文章，
讲清楚 AI 改变了什么、测试工程师要补什么能力。
```

## 已知信息

- **主题关键词**：AI、测试工程师、职业发展、能力升级
- **方向关键词**：AI 改变了什么、要补什么能力
- **用户没给的信息**（agent 主动推断，不反问）：
  - 栏目
  - 历史文章库
  - 字数要求
  - 目标读者画像
  - 系列规划倾向
  - 配图需求

## 给 agent 的处理建议（不在原文中，外部注释）

按 `rules/writing-topic-selection.md` 的 7 项逐条输出：

1. **与旧文重复风险评估** — 标注"未提供历史文章库，基于常见公众号选题进行推断"
2. **推荐新切入角度** — 至少 2 个，避开"AI 是工具"这种泛泛切入
3. **方法论 vs 实战** — 给出判断和理由
4. **是否适合系列** — 给出 3~5 期递进规划
5. **推荐栏目** — 按"无栏目库默认推断"给 `质量内功`（也可并列 `AI 落地手记`）
6. **读者画像** — 限定岗位 / 年限 / 痛点场景
7. **最容易踩的坑** — 内容层面，不是技术层面

**不要**反问"请提供栏目库 / 历史文章"。**不要**在用户确认前写正文。
