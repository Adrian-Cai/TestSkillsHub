# 回复边界

## 总原则

这是一个"文件交付型"阶段，不是"代码讲解型"。

当本阶段适用时：

- 用户可见回复中不得输出 HTML / JS / CSS 源码
- 中途回复也不得用"先给你一版代码"替代实际文件
- 只有在用户明确要求检查已存在文件内容时，才可以展示文件片段

## 最终回复形状

必须严格使用以下结构：

1. 第一行：`.html` 文件链接或路径
2. 后续五行：五步极简使用说明

除此之外，不得添加：

- 标题
- 标签
- 解释性段落
- 致谢或结束语

## 常见偏离

如果你发现自己正在做下面的事，说明已经偏离：

- 先贴部分代码"方便用户确认"
- 先输出 HTML 模板，再说稍后生成文件
- 说"复制下面代码保存成 html 即可"
- 在最终答案里加上"已生成如下"之类的引导语
