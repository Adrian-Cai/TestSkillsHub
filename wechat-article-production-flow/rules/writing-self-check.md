# 输出前自检清单

在输出正文前，逐条过一遍。每项是决策性 checkbox，不是装饰。

## 硬约束（任一未通过都必须重写）

- [ ] 字数在 2500 ~ 3500 之间？超 3500 = REFACTOR 重写，不是在文末砍字数
- [ ] 一级标题 ≤ 5 个？每章只讲 1 个核心观点？违反 = 拆章节
- [ ] 连续纯文字段 < 400 字？违反 = 在该段前插小标题 / 列表 / 加粗 / 引用
- [ ] 一级标题数量 = Phase 2 配图数 = 6？违反 = 触发补足或压缩策略
- [ ] 开头 3 句话内抛出钩子？无钩子 = 重写开头

## 风格约束（任一未通过都需要 REFACTOR）

- [ ] 通篇读起来像"技术人在分享"，而不是"AI 在交作业"？
- [ ] 每段包含真实场景 / 真实风险 / 可落地动作之一？
- [ ] 0 个"AI 不是 A 而是 B"排比收尾？
- [ ] 0 个"综上所述""本文旨在"论文腔？
- [ ] 0 个"测试很重要"型废话？

## 产物约束

- [ ] 末尾有"排版注意点"提醒？
- [ ] 文末有"下一篇我们聊 XX"钩子（可选但推荐）？

## 触发任一未通过

不要"先发出去再改"。回到对应章节重写那一段；连续 2 次未过，回到文章策划重出大纲。
