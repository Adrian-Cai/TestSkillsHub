# 选题判断

等用户提供「本期输入」后，逐条输出以下 7 项：

1. **与旧文重复风险评估**（高/中/低）
   - 列出具体哪几篇最接近
   - 若用户未提供历史文章：明确标注"未提供历史文章库，基于常见公众号选题进行推断"并给出推断逻辑
   - 若用户引用了一篇不存在的旧文：标注"未找到该文章记录"，**不编造旧文标题或细节**
   - 若重复风险为中或高，必须主动替换切入角度/核心矛盾/结构/案例/输出重点中至少 3 项，并说明替换逻辑

2. **推荐的新切入角度**（2~3 个，标注差异点）

3. **本期更适合"方法论"还是"实战"**，并说明原因

4. **是否适合开成系列**
   - 若适合，给出 3~5 期的递进规划

5. **推荐栏目**
   - 若用户已指定栏目：直接采用
   - 若用户未指定栏目：按下方"无栏目库默认推断"给出 1~2 个推断栏目，并标注"默认推断，可被覆盖"
   - **不要反问"请提供栏目库"**

6. **最容易引发共鸣的读者画像**
   - 具体到岗位 / 年限 / 痛点场景

7. **本期最容易踩的坑**
   - 内容上的，不是技术上的

输出后等待用户确认，说"继续"才进入文章策划。

## 无栏目库默认推断

按以下三类主题方向各配一个默认栏目（可被覆盖）：

- **AI 测试 / Agent 测试方向**：默认栏目 `AI 落地手记`
- **测试开发 / 自动化方向**：默认栏目 `工程实战`
- **质量工程 / Diff 风险 / 流程治理方向**：默认栏目 `质量内功`

若主题跨多个方向，给出两个候选栏目，按占比排序。
