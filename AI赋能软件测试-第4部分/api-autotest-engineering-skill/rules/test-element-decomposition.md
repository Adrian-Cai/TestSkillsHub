# 测试要素拆解规范

## 目标

从接口资料中系统性地拆出所有测试要素，确保测试覆盖不遗漏关键维度。

## 六个必覆盖维度

### 1. 参数校验

针对每一个请求参数，检查以下测试点：

| 测试点类型 | 说明 | 适用场景 |
| ---------- | ---- | -------- |
| 必填参数缺失 | 不传必填参数 | 所有必填参数 |
| 参数为空字符串 | 传空字符串 `""` | string 类型参数 |
| 参数为 null | 显式传 null | 所有参数 |
| 类型错误 | 传错误类型（数字传字符串等） | 有类型约束的参数 |
| 长度边界-最小值 | 传最小允许长度的值 | 有长度限制的参数 |
| 长度边界-最大值 | 传最大允许长度的值 | 有长度限制的参数 |
| 长度超限 | 传超过最大长度的值 | 有长度限制的参数 |
| 格式错误 | 传不符合格式要求的值（邮箱、手机号等） | 有格式约束的参数 |
| 非法特殊字符 | 传 SQL 注入、XSS、特殊符号 | 字符串输入参数 |
| 超大数字 | 传极大整数或浮点数 | 数值类型参数 |
| 负数 | 传负数 | 非负数值参数 |

### 2. 业务规则

从接口的业务逻辑中拆出约束条件：

| 规则类型 | 说明 |
| -------- | ---- |
| 状态限制 | 某状态下不允许调用（如已注销用户不能登录） |
| 唯一性约束 | 重复数据不允许创建（如重复注册同一邮箱） |
| 重复提交 | 同一请求重复调用的幂等行为 |
| 权限边界 | 不同角色能操作的数据范围 |
| 数量限制 | 单次操作的数量上限 |
| 金额限制 | 最小/最大金额约束 |
| 时间限制 | 有效期、时间窗口 |
| 依赖关系 | 需要先完成某操作才能调用 |

### 3. 返回校验

每个接口的返回结果必须覆盖：

| 校验项 | 说明 |
| ------ | ---- |
| HTTP 状态码 | 200/201/400/401/403/404/500 等 |
| 业务 code 字段 | 成功/失败的业务码（不能只看 HTTP 200） |
| message 字段 | 错误描述是否符合预期 |
| 核心业务字段 | 成功时必须返回的字段（如 token、id、data） |
| 字段类型 | 返回字段的数据类型是否正确 |
| 敏感字段 | 失败场景不应返回密码、token、敏感数据 |

### 4. 鉴权校验

鉴权相关测试点（适用于需要鉴权的接口）：

| 测试点 | 说明 |
| ------ | ---- |
| 无 token | 请求头不携带 token |
| token 为空 | token 字段存在但值为空 |
| token 格式错误 | 传乱码或错误格式的 token |
| token 已过期 | 使用已过期的 token |
| token 对应用户已注销 | token 有效但用户状态异常 |
| 权限不足 | 普通用户调用管理员接口 |
| 越权访问 | 访问他人资源 |

### 5. 数据状态

测试不同数据状态下接口的行为：

| 数据状态 | 说明 |
| -------- | ---- |
| 正常数据 | 标准的可用数据 |
| 不存在的 ID | 查询/操作一个不存在的资源 |
| 已删除数据 | 操作已被软删除的数据 |
| 已禁用数据 | 操作被禁用状态的资源 |
| 边界数量数据 | 列表接口的空列表、单条、大量数据 |
| 脏数据 | 历史遗留的异常数据 |

### 6. 安全风险

| 风险类型 | 检查点 |
| -------- | ------ |
| 敏感信息泄露 | 错误响应中是否包含密码、内部路径、堆栈信息 |
| 越权操作 | A 用户能否操作 B 用户的数据 |
| 错误提示过细 | 错误信息是否暴露内部实现（如「用户不存在」vs「用户名或密码错误」） |
| 参数注入 | SQL 注入、XSS、命令注入 |
| 大量请求 | 接口是否有频率限制 |

## 输出要求

- 为每个维度列出适用于当前接口的具体测试要素。
- 不适用的维度注明「不适用，原因：XXX」。
- 无法确认的业务规则标注「待确认」，不猜测。
- 每个测试要素后续必须对应测试点矩阵中的至少一条记录。
