## 优惠券案例优化前后对比样例

### 案例需求简述

用户可以领取平台优惠券，并在下单时使用。每个用户每种优惠券限领一次。优惠券有使用门槛、有效期、适用商品范围。订单取消后优惠券是否退回由优惠券规则决定。过期、已使用、未达到门槛、商品不适用时不可使用。

---

### 第一轮：直接生成后的典型结果

| 用例编号 | 用例标题 | 主要问题 |
|---|---|---|
| TC-001 | 用户成功领取优惠券 | 只覆盖主流程 |
| TC-002 | 用户下单使用优惠券 | 未说明门槛、商品范围、有效期 |
| TC-003 | 用户重复领取优惠券 | 有异常，但缺少预期状态校验 |
| TC-004 | 优惠券过期不可使用 | 未覆盖边界时间 |
| TC-005 | 优惠券使用后状态变更 | 未覆盖订单取消后的状态 |

### 第一轮问题总结

| 问题类型 | 表现 |
|---|---|
| 规则拆解不足 | 没有系统拆出领取限制、使用门槛、适用范围、退券规则 |
| 边界不足 | 没有覆盖刚好达到门槛、刚好过期、有效期最后一秒 |
| 条件组合不足 | 没有覆盖金额、商品、有效期、状态的组合 |
| 状态不足 | 没有覆盖未领取、已领取、已使用、已过期、已退回 |
| 风险不足 | 没有突出重复使用、金额错误、资损风险 |

### 第二轮：按测试设计方法优化后的样例

| 用例编号 | 方法 | 用例标题 | 测试数据 | 预期结果 | 优先级 |
|---|---|---|---|---|---|
| TC-Coupon-001 | 场景法 | 用户首次领取有效优惠券 | 用户未领取，优惠券未过期 | 领取成功，券状态为已领取 | P0 |
| TC-Coupon-002 | 场景法 | 用户领取后下单成功使用优惠券 | 订单金额满足门槛，商品适用 | 优惠金额正确抵扣，券状态为已使用 | P0 |
| TC-Coupon-003 | 等价类 | 已领取用户再次领取同一优惠券 | 同一用户重复领取 | 领取失败，提示已领取 | P1 |
| TC-Coupon-004 | 等价类 | 商品不在适用范围时不可使用 | 商品不适用 | 优惠券不可选或使用失败 | P1 |
| TC-Coupon-005 | 边界值 | 订单金额刚好等于使用门槛 | 门槛 100，订单 100 | 可以使用优惠券 | P0 |
| TC-Coupon-006 | 边界值 | 订单金额低于使用门槛 0.01 | 门槛 100，订单 99.99 | 不可使用优惠券 | P1 |
| TC-Coupon-007 | 边界值 | 优惠券在有效期最后一刻使用 | 当前时间等于截止时间前最后有效点 | 可以使用优惠券 | P1 |
| TC-Coupon-008 | 边界值 | 优惠券过期后使用 | 当前时间超过有效期 | 不可使用优惠券 | P1 |
| TC-Coupon-009 | 判定表 | 金额满足、商品适用、券未过期、券未使用 | 全部条件满足 | 使用成功 | P0 |
| TC-Coupon-010 | 判定表 | 金额满足但商品不适用 | 商品条件不满足 | 使用失败 | P1 |
| TC-Coupon-011 | 状态迁移法 | 已领取优惠券使用后变为已使用 | 已领取券完成支付 | 状态从已领取变为已使用 | P0 |
| TC-Coupon-012 | 状态迁移法 | 已使用优惠券不可再次使用 | 券状态为已使用 | 不可重复使用 | P0 |

### 第三轮：AI 自查遗漏后补充的异常场景

| 遗漏场景 | 补充原因 | 是否纳入 | 优先级 |
|---|---|---|---|
| 订单取消后优惠券是否退回 | 涉及状态回退和权益恢复 | 需确认规则后纳入 | P1 |
| 支付失败后优惠券是否释放 | 涉及券锁定和订单状态一致性 | 建议纳入 | P1 |
| 并发使用同一张优惠券 | 涉及重复抵扣和资损 | 建议纳入 | P0 |
| 优惠券金额计算精度 | 涉及金额准确性 | 建议纳入 | P0 |
| 优惠券与其他活动叠加 | 涉及复杂业务规则 | 需确认规则后纳入 | P1 |
| 用户无权限领取指定券 | 涉及权限和定向发券 | 视业务是否支持 | P1 |

### 人工裁剪与合并后的最终样例

| 最终编号 | 用例标题 | 覆盖目标 | 来源 | 处理方式 | 优先级 |
|---|---|---|---|---|---|
| FC-001 | 首次领取有效优惠券成功 | 领取主流程 | TC-Coupon-001 | 保留 | P0 |
| FC-002 | 满足门槛且商品适用时成功使用优惠券 | 使用主流程、金额抵扣 | TC-Coupon-002、009 | 合并 | P0 |
| FC-003 | 重复领取同一优惠券失败 | 领取限制 | TC-Coupon-003 | 保留 | P1 |
| FC-004 | 订单金额边界校验 | 等于门槛、低于门槛 | TC-Coupon-005、006 | 合并为参数化用例 | P0 |
| FC-005 | 优惠券有效期边界校验 | 最后有效点、过期后 | TC-Coupon-007、008 | 合并为参数化用例 | P1 |
| FC-006 | 商品不适用时不可使用优惠券 | 商品适用范围 | TC-Coupon-004、010 | 合并 | P1 |
| FC-007 | 已使用优惠券不可重复使用 | 状态与重复使用风险 | TC-Coupon-011、012 | 合并 | P0 |
| FC-008 | 并发使用同一张优惠券只成功一次 | 并发与资损风险 | 自查补充 | 新增 | P0 |
| FC-009 | 订单取消后优惠券状态按规则处理 | 状态回退 | 自查补充 | 需确认后纳入 | P1 |
| FC-010 | 支付失败后优惠券释放或保持锁定符合规则 | 订单与券状态一致性 | 自查补充 | 新增 | P1 |

### 优化前后差异

| 对比项 | 优化前 | 优化后 |
|---|---|---|
| 生成方式 | 直接生成 | 规则拆解 + 方法生成 + 反向检查 |
| 覆盖结构 | 主流程为主 | 主流程、异常、边界、组合、状态均覆盖 |
| 用例质量 | 重复、泛化、粒度不一 | 可执行、可评审、可排序 |
| 风险覆盖 | 资损和并发风险不突出 | P0 场景突出金额、重复使用、并发 |
| 人工参与 | 输出后被动查看 | 全程审查、裁剪、合并、确认 |
